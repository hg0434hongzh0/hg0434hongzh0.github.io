WAF的一般原理

  • 监控和过滤传入流量
  • WAF使用预定义的规则集以及正则匹配来判断流量
  • 高级WAF能够进行行为分析，检测异常活动，如用户请求频率、请求头、UA等
  • 拦截恶意流量
  • 一些现代的 WAF 具有学习功能，能够自适应地调整其规则以适应新的攻击模式
  • WAF 通常会记录所有传入流量的活动，并生成报告
  • WAF 的一般原理是通过监视、分析和过滤传入的网络流量，以识别和阻止各种类型的网络攻击，从而保护 Web 应用程序的安全性

HTTP头部的server字段的值有哪些信息

  • 一般包括服务器软件的相关信息——服务器名称+版本号
  • 可能出现Apache/IIS/Nginx 以及版本 以及操作系统
  • 这项字段一般隐藏，否则不安全

Log4j2反序列化原理

  • log4j2是一个底层日志组件，它的lookup接口会造成通过JNDI引用命名服务
  • log4j2记录日志的时候，碰到${xxx}这样的文字没有一比一地记录下来，而是去解析
  • 可以通过JNDI接口引用到DNS/RMI/LDAP协议接口
  • 攻击者可以自启一个服务器，通过${}传入精心构造的payload让目标服务器去请求攻击者构造的LDAP服务器，以及攻击者自定义的内容
  • 攻击者的服务器就会自动生成恶意的Java类，然后通过HTTP端口下发给目标服务器
  • 目标服务器收到之后直接加载到内存中执行，造成RCE

未授权访问原理

  • 需要身份认证的网站应用/目录或是服务器组件等在未经身份认证的情况下就被成功访问
  • 服务器可能缺乏访问控制，文件权限/ACL/数据库权限错误配置
  • 系统、服务或资源本身存在安全漏洞

XXE漏洞原理

  • XXE就是服务器对XML解析时没有禁掉外部实体的引用与解析
  • 攻击者构造的DTD声明外部实体的XML文档可以成功传入服务器并被解析
  • 可能造成文件读取、信息泄露、RCE等

Weblogic SSRF漏洞（CVE-2014-4210）利用过程

  1. 问题出在SearchPublicRegistries.jsp下的operator参数，其参数值可以请求服务器内部资源
  2. SSRF探测内网ip:6379确定Redis存在
  3. SSRF利用Gopher协议操纵Redis，写入计划任务，执行恶意代码
  4. 需要结合Weblogic的特性，使用CRLF（%0a%0d）注入换行符，确保命令执行

IIS中间件PUT文件写入的修复建议

  • 禁用PUT方法
  • 使用PUT方法时，对身份进行验证
  • 通过ACL禁止未授权用户写入
  • 对文件名和内容进行输入验证
  • 隔离上传目录，不返回路径
  • 采用文件扩展名白名单

Struts2攻击特征

  • RCE试图传入命令及执行结果
  • 伪造参数，尝试OGNL表达式
  • 典型URL后缀：.do、.action、.jsp
  • 能够处理OGNL表达式

Fastjson原理

  • 服务器解析JSON时未对@type过滤，导致攻击者可构造恶意类TemplateImpl
  • 该类的_bytecodes字段可控，实例化时导致RCE
Fastjson版本变化
  • 1.2.25：@type黑名单，需使用Lcom.sun.rowset.JdbcRowSetImpl;
  • 1.2.42：更严格黑名单检测，需使用LLcom.sun.rowset.JdbcRowSetImpl;;
Fastjson组件判断
  • 请求是否以JSON格式传值，且内容包含@type
  • 通过提交特殊字符获取错误信息，判断是否调用了JSON解析

Shiro特征与漏洞

  • rememberMe字段存在于Cookie中
  • 登录失败返回rememberMe=deleteMe
  • Shiro550 vs Shiro721
    • 550可直接跑默认key
    • 721默认key无效，需获取有效cookie
    • 利用方式
      1. rememberMe字段被AES加密+Base64编码
      2. 服务器解密并反序列化
      3. 若密钥可控，可利用ysoserial发送恶意payload，通过CC链执行命令

WAF & 天眼联动导致IP变化

  • WAF先拦截和过滤请求，天眼再分析
  • 因此天眼看到的源IP是WAF，而非真实攻击者IP

DNS设备出现木马告警的可能性

  • 设备被木马感染
  • 设备被攻击者利用（如DNS劫持）
  • 误报
  • 需要检查DNS设备日志、流量数据，确保安全

命令执行告警的误报判断

  • 返回包是否有命令执行结果
  • 核实命令来源，是否是正常业务
  • 命令内容是否包含危险指令

负载均衡环境下如何找源IP

  • 代理头（XFF/X-Real-IP）
  • 负载均衡日志
  • Wireshark分析流量

Windows疑似被挖矿的排查思路

  • 检查系统资源、CPU占用
  • netstat -anp查找异常端口进程
  • wmic分析进程参数
  • 检查启动项、自启脚本
  • 扫描恶意软件
  • 分析网络流量，检查是否有连接挖矿池
  • 审查系统日志
  • 检查防火墙规则
  • 直接查找挖矿程序

Webshell管理工具流量特征分析

1. 菜刀

  • 采用一句话木马
  • payload 明文传输，使用 URL + base64
  • payload 包含 eval / assert / base64_decode
  • payload 默认固定 &z0=QGluaV9zZXQ 开头，base64 解密后可见代码

2. 蚁剑

  • 使用 assert / eval，带有 base64 编解码
  • 请求体开头特征：@ini_set("display_errors","0");@set_time_limit(0);
  • payload 经过混淆后仍可解码，格式：随机数 + 结果 + 随机数
  • 参数名 _0x 开头，例如 _0x...=

3. 冰蝎

  • 冰蝎2.0：AES + base64，动态密钥，UA 头频繁切换
  • 冰蝎3.0：AES + base64，固定密钥，UA 头频繁切换
    • Content-Type=application/octet-stream
    • 关键特征密钥：e45e329feb5d925b
  • 冰蝎4.0
    • UA 头频繁切换
    • 请求使用 Connection: Keep-Alive
    • 特定端口，如 49700
    • 特定请求/响应头：
      • 请求字节头：dFAXQV1LORcHRQtLRlwMAhwFTAg/M
      • 响应字节头：TxcWR1NNExZAD0ZaAWMIPAZjH1BFBFtHThcJSlUXWEd

4. 哥斯拉

  • webshell 动态生成
  • JSP 中出现 xc / pass 字符、Java 反射 (ClassLoader)
  • base64 编解码
  • UA、Accept 字段较弱（可修改）
  • Cookie 末尾带 ;
  • 响应体特征：
    • 32 位 MD5 拆分：md5前16位 + base64 + md5后16位

域控被攻陷

  • krbtgt hash 变更，防止黄金票据
  • ACL 策略修复
  • 检查 域控伪造
  • SSP 注入
  • DCSync 高权限用户攻击
  • 检查：
    • 约束性委派
    • 非约束性委派
    • 基于资源的委派

  • 确认 192.168.1.2 是否能直通 3.4
  • 若不通，检查 多网卡跳板

常见安全知识

TCP 三次握手

  1. SYN 发送连接请求
  2. SYN-ACK 服务器确认
  3. ACK 客户端确认

X-Forwarded-For (XFF)

  • 获取用户真实 IP

HTTP 状态码

  • 401 认证失败
  • 403 中间件拒绝
  • 301 永久重定向
  • 500 服务器错误

SQL 注入

  • 二次注入：数据存入数据库，后续利用
  • 堆叠注入：; 执行多个 SQL 语句
  • 读写条件：
    • secure_file_priv=''（空或 NULL）

SQL 注入防御

  • 预编译
  • PDO
  • WAF 过滤

XSS vs CSRF

  • XSS：跨站脚本，攻击用户
  • CSRF：跨站请求伪造，攻击服务器

XXE 盲注

  • 外带数据回显
